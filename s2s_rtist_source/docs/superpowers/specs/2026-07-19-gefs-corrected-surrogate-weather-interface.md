# 冻结 GEFS 降水订正接入代理模型天气接口

## 目的

本阶段只验证数据接口，不训练代理模型、不运行 SWAP、不重新拟合天气订正。输入为现有
31 成员 GEFS 逐日天气，降水通过冻结 artifact 订正，其他气象变量原样保留。

## 两层输出

1. 成员逐日表：保留 31 成员、7 天、raw/订正降水及所有原始天气变量，用于审计与未来
   概率决策。
2. 代理模型宽表：每站点-起报周期一行；逐日非降水变量使用集合均值，降水同时保留 raw
   均值、订正均值、订正 P10/P90。该表可按 `site_id + decision_date` 连接到每个 0-60 mm
   连续灌溉候选。

## 单位映射

- GEFS `shortwave_w_m2` 转为 SWAP `Solar`：乘 `86.4` 得 `kJ/m2/day`。
- 温度保持摄氏度，风速保持 `m/s`，降水保持 `mm/day`。
- GEFS 提供的是 VPD，不能直接写入 SWAP `HUM`。先以日均温计算饱和水汽压，再使用
  `actual_vapor_pressure=max(0,es-vpd)` 得到 SWAP 所需实际水汽压 `kPa`。

## 标签与物理约束

正式代理输出为未来 7 天净收益、累计实际蒸散发和逐日固定 0-100 cm 土壤体积含水率。
当前 SWAP 版本的实际蒸散发必须包含 `Tact + Eact + Interc`。水量平衡辅助字段继续保留
降水、灌溉、径流、固定土层排水、100 cm 边界流和储水变化，用于预训练及 TTA 物理损失。

本 smoke 只验证天气侧字段、单位、日期、成员数、artifact 追踪和非降水字段零改动。
