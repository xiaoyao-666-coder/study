# GEFS QDM 与扩展地面站降水设计规范 v1

## 1. 目的

本阶段响应老师的两项建议：

1. 使用相应站点的真实地面天气观测扩展矫正模块的训练年份；
2. 尝试 Cannon、Sobie 与 Murdock（2015）提出的 Quantile Delta Mapping（QDM）。

本阶段只做数据源审计、少量站点文件提取、QDM 离线验证和 2019 验证设计，不下载 GHCN-Daily 全库，不启动代理模型训练，不修改 TTA。

## 2. 为什么改变现有方案

2015-2018 的 ERA5 参考数据只有 840 个独立站点-日期观测。训练期 OOF 结果显示，普通 QM 的细分组容易过拟合；即使 `site_only` 和全局映射改善部分均值指标，也会缩窄五成员集合并降低重事件覆盖率。扩大独立年份和改用地面站观测，分别针对“样本太少”和“格点再分析不等于站点实测”两个问题。

## 3. 数据范围

### 3.1 GEFS

- 使用 NOAA GEFSv12 reforecast 的同质化公开档案期 `2000-2019`。
- 每日 `00 UTC` 起报，成员固定为 `c00, p01, p02, p03, p04`。
- 每个起报周期提取未来 7 天逐日降水。
- 保留现有六个生长季起报日：`06-01, 06-15, 07-01, 07-15, 08-01, 08-15`。
- `2000-2018` 为矫正模型训练与内部时间分块验证范围，`2019` 只作验证，`2024` 不参与本阶段拟合或方法选择。

### 3.2 地面站观测

- 主源：NOAA NCEI GHCN-Daily，变量 `PRCP`。
- 原始 `PRCP` 单位为 `0.1 mm`，读取后乘 `0.1` 转为 `mm/day`。
- 质量标志 `Q_FLAG` 非空的记录不得进入拟合或验证。
- 测量标志、来源标志必须原样保留，不能只保存转换后的数值。
- 不允许用 ERA5 或 gridMET 静默填补站点缺测。缺测必须保留并计入完整率。

## 4. 站点匹配规则

对 P1、P2、P3、P4、P15 分别执行：

1. 从 GHCN-D inventory 中筛选 `PRCP` 记录覆盖 `2000-2019` 的站点；
2. 初始搜索半径为 `100 km`，输出最近 10 个候选；
3. 下载候选站自身的 `by_station/<station_id>.csv.gz`，不下载全库；
4. 在项目所需的 840 个站点-有效日期上统计：存在率、无质量标志率、逐年完整率和最长连续缺测；
5. 正式主站必须达到总体无质量标志完整率 `>=95%`，且每年完整率 `>=85%`；
6. 多个候选同时合格时选择距离最近者；没有候选合格时才将半径扩到 `150 km`，并单独报告空间代表性风险；
7. 每个项目站点只使用一个固定主站。不得按天混合不同站点来提高完整率。

站点日降水的观测日界可能不是 UTC 00-24。第一版不自动平移日期；必须在 `2000-2018` 内额外报告 `-1/0/+1 day` 配对诊断。只有训练期证据明确且预先记录后，才能锁定站点日期偏移；不得用 2019 或 2024 选择偏移。

## 5. QDM 数学定义

对目标时期 GEFS 降水 `x_m,p(t)`：

1. 目标时期非超越概率

   `tau_m,p(t) = F_m,p^(t)(x_m,p(t))`

2. 同一分位数上的模式相对变化

   `Delta_m(t) = x_m,p(t) / F_m,h^(-1)(tau_m,p(t))`

3. 历史观测分布上的偏差订正值

   `x_hat_o:m,h:p(t) = F_o,h^(-1)(tau_m,p(t))`

4. 降水 QDM 输出

   `x_hat_m,p(t) = x_hat_o:m,h:p(t) * Delta_m(t)`

乘法形式用于保留降水分位数的相对变化。若目标分布与历史模式分布相同，QDM 退化为普通 QM。

## 6. 零降水和可重复性

Cannon 等（2015）将低于 `0.05 mm/day` 的值作为 trace：订正前把零值替换为 `(0, 0.05)` 内的均匀随机数，订正后小于 `0.05 mm/day` 的值重新设为零。本项目保留这一原则，但固定随机种子，并按稳定行键排序后生成扰动，保证相同输入得到相同结果。

第一版不设置相对变化比值上限，也不对上尾静默裁剪；必须输出最大比值、非有限值数、负值数和极端订正值，若出现数值不稳定则候选直接失败。

## 7. 候选集合

老师已明确只比较按站点和全局两种分组，因此预锁定候选为：

- `raw_gefs`：原始 GEFS 基线；
- `qm_global`：现有经验 QM，全局一个映射；
- `qm_site_only`：现有经验 QM，每站点一个映射；
- `qdm_global`：QDM，全局历史/目标分布；
- `qdm_site_only`：QDM，每站点历史/目标分布。

不再评估 `lead_only`、`site_lead` 或月份细分组。

## 8. 离线验证与实时边界

论文中的 `F_m,p^(t)` 需要目标时期模式样本。当前第一版使用完整留出年份的 GEFS 预报来估计目标 CDF：它不使用该年的地面观测拟合，因此可用于离线方法比较，但会看到该年后续 GEFS 样本，不是严格实时算法。

执行顺序：

1. 在 `2000-2018` 内做按年份或连续年份块的 OOF；
2. 只根据训练期 OOF 锁定 `global` 或 `site_only` 及 QM/QDM 方法；
3. 冻结选择后运行 2019 验证；
4. 只有 QDM 在训练期和 2019 都有稳定收益，才另立“因果滚动目标 CDF”设计；
5. 因果版本通过前，不允许把离线 QDM 用于 2024 实时 TTA。

## 9. 评价与晋级 gate

沿用现有证据体系：

- 主要指标：7 天累计降水 MAE、CRPS、`1/5/10/20 mm` 平均 Brier；
- 概率可靠性：重事件 P10-P90 覆盖率和成员 min-max 覆盖率；
- 稳定性：逐年/时间块不劣于 raw 的次数；
- 数值审计：负值、非有限值、最大 QDM 比值、最大订正降水；
- 数据审计：主站距离、完整率、质量标志、缺测和日期偏移敏感性；
- 泄漏审计：留出年份地面观测不得进入历史 CDF，2019/2024 不得参与候选选择。

任何候选若改善均值但同时使重事件两种覆盖率均下降，仍不晋级。QDM 的目标不是放松原 gate，而是检验“保留 GEFS 分布变化”能否缓解普通 QM 的集合收缩。
