# GEFS 全季主干与七日独立分支标签生成设计

## 已确认目标

正式训练标签不再从孤立日期分别构造预状态。每个站点、每个年份先由 ERA5 从播种到收获连续驱动 SWAP，形成一条不受候选灌溉分支影响的季节主干。决策分支始终从主干状态出发，只运行未来七天，结束后丢弃。

## 决策日规则

- SWAP 分支从决策日当天开始，所需 restart checkpoint 是决策日前一天结束时的状态。
- 在主干 checkpoint 的 `DVS >= 0.1` 后开始采样。
- 第一个决策日是首次满足阈值的 checkpoint 次日。
- 后续按每七个日历日采样一次，各站点独立生成日期，不要求五站点同日对齐。
- 七日窗口采用决策日到决策日加六天的闭区间，窗口末日不得晚于收获日。
- 主干作物输出必须逐日连续；缺日、重复日期、DVS 缺失或七日窗口不完整均为硬失败。

## 天气协议

主干始终使用对应年份 ERA5。每个分支未来七天必须将 SWAP 所需六个字段全部替换为无年份泄漏的矫正 GEFS：

1. 降水；
2. 最低温；
3. 最高温；
4. 实际水汽压；
5. 风速；
6. 太阳辐射。

当前只有降水矫正已经冻结。其余五个字段原始 GEFS 透传不符合正式训练协议，因此在完成参考数据、矫正方法和逐年 OOF 验证前，分支批量生成保持阻断。

## 主干状态技术门槛

SWAP 的 `.end` 文件只保存一次运行结束状态，不能直接假定一次完整季节运行已经产生所有决策日 checkpoint。正式实现前先在一个站点、一个年份比较两种可行方案：

- 从播种开始，分别运行到每个 checkpoint；
- 从前一个 checkpoint 继续串联 restart 到下一个 checkpoint。

两者还需与单次连续季节模拟的同日 `.crp`、土壤状态和累计量核对。只有状态一致性通过后，才选择服务器成本更低的 checkpoint 生成方式。

## 阶段顺序

1. 用完整季节 `.crp` 生成 DVS/收获约束下的决策日计划。
2. 做一个站点、一个年份的主干 checkpoint 等价性 smoke。
3. 为五个非降水变量锁定参考数据、无泄漏矫正方法和门槛。
4. 做一个站点、一个决策日、八种灌溉量的全变量分支 smoke。
5. 扩到五站点、一个年份，再决定是否生成 2015-2019 全量标签。

任何阶段均不训练代理模型，也不执行 TTA。
