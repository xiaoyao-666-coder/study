# 订正 GEFS 驱动 SWAP 三输出 Smoke 规范

## 目标

用冻结的 GEFS 降水订正模块和同周期 31 成员日集合均值，真正驱动一次小规模 SWAP 标签生成：

- 日期：`2024-07-16`；
- 站点：P1、P2、P3、P4、P15；
- 灌溉候选：`0,10,15,20,25,30,40,60 mm`；
- 规模：5 站点 x 8 候选，共 40 行；
- 输出：7 天增量净收益、7 天累计 AET、未来 7 天逐日固定 0-100 cm 土壤含水率及水量平衡辅助量。

本次不下载数据、不拟合天气订正模块、不调参、不训练代理模型。

## 因果天气边界

每个站点从已确认工作区复制出独立运行副本：

1. `2024-07-15` 及以前继续使用原工作区 gridMET，生成决策日前状态；
2. 只把 `2024-07-16` 至 `2024-07-22` 替换为当前起报周期可见的订正 GEFS 天气；
3. 同时更新 `weather.024`、`WeatherOriginal.024` 和 `df_gridmet.csv`；
4. SWAP 实际读取 `weather.024`；另外两个文件用于防止工作副本内部口径不一致；
5. 每个被替换日期的旧值、新值、来源、artifact hash 和文件哈希都必须保留。

因此，未来 GEFS 不得进入决策日前状态模拟。

## 天气映射

| SWAP 字段 | 订正 GEFS 集合日统计 |
|---|---|
| RAD | `solar_kj_m2_day_mean` |
| Tmin | `temperature_min_c_mean` |
| Tmax | `temperature_max_c_mean` |
| HUM | `actual_vapor_pressure_kpa_mean` |
| WIND | `wind_speed_m_s_mean` |
| RAIN | `precipitation_mm_corrected_mean` |

工作区已核对为 `SWETR=0`，SWAP 不读取天气文件中的 `ETref` 作为参考蒸散输入，而是根据辐射、温度、实际水汽压和风速计算。注入时保留原 `ETref` 数值仅用于维持文件格式。

这一步使用单一的集合均值天气情景。集合成员级不确定性传播留到情景一致 smoke 通过后单独设计。

## 通过标准

- 5 个站点全部正常完成，产生 40 行候选；
- 每个工作副本的 3 个天气文件各替换恰好 7 天；
- 决策日前天气修改行数为 0；
- 每站点 SWAP `rain_7d_mm` 与输入订正 GEFS 7 天降水和相差不超过 `0.01 mm`；
- 三输出无缺失，AET 恒等式和固定控制体字段通过原正式验证器；
- 最大绝对水量平衡残差不超过 `0.5 mm`；
- 灌溉候选位于 `0-60 mm`，且 0 mm 与 60 mm 原始输出被保留；
- 输出明确写明 `weather_label_scenario_consistent=true`；
- 输出仍写明 `training_eligible=false`，因为只有一个日期，不能当训练集。

## 通过后的含义

通过只说明“订正 GEFS -> SWAP -> 三输出标签”的单日期链路成立。随后应先整理 2015-2019 批量生成设计、样本量和存储预算，再决定是否启动正式数据生成；不得在本次 smoke 后直接训练模型。
